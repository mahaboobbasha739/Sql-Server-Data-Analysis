						Indexes
						------------
	Indexes are used to Retrieve / get the Data from the tables very quickly
	Indexes are used to Improve the Performance of the Queries

	The Default Search Process is "Sequential Search"

	Select * from Emp where Esalary Between 20000 and 40000

	Types of Indexes
	-----------------------
	1.Clustered Index
	2.Non-Clustered Index
	3.Unique Index
	4.Composite Index

	1.Clustered Index
	-------------------------
	Clustered Index are used to Rearrange the Data in the Table either in Asc / Desc
	Only one Clustered Index for one table
	P.K will Automatically creates an Clustered Indes
	Both the Index and Data is Stored at one Place

	4.Composite Index
	--------------------------
	Creating the Clustered Index Combindelly for Multiple Columns is known as "Composite Index"

	2.Non-Clustered Index
	-------------------------------
	In Non-Clustered Indexes the Data is Stored at one Palce and the Index is Stored at another Place
	The Non-Clustered Indexes will have the Pointer to Store the Location of Dats
	We can Create multiple Non-Clustered Indexes for Multiple Columns in a table because the Data is Stored at one Palce and the Index is Stored at another Place
	
	3.Unique Index
	--------------------
	Unique Constarint  --> Unique Non-Clustered Index
	If the Column doesn't contains the Duplicate values on those Columns only we are able to Create the Unique Index
